# QueueBus 1페이지 요약서

## 서비스명

QueueBus

## 한 줄 소개

QueueBus는 정류장 반경 내 위치 인증으로 광역버스 노선별 가상 대기열을 만들고, AI가 다음 버스 승차 가능성과 호출 타이밍을 예측해 이용자가 줄 대신 대기번호로 편하게 기다릴 수 있게 하는 LBS 기반 대중교통 대기 관리 서비스입니다.

## 문제

광역버스 정류장의 물리적 줄서기는 출퇴근 시간대에 장시간 대기를 만들고, 폭염·한파·우천 노출, 보행로 점유, 새치기 갈등, 탑승 가능성 불확실성, 운영 데이터 부족 문제로 이어집니다. 이용자는 이번 차를 탈 수 있는지 몰라 계속 줄을 설 수밖에 없고, 지자체와 운수사는 정류장별·노선별·시간대별 실제 대기 수요를 정량적으로 파악하기 어려워 민원과 체감에 의존해 혼잡 정류장을 관리하는 경우가 많습니다.

## 해결

QueueBus는 정류장 근처에 실제 도착한 승객만 위치 인증 후 노선별 가상 대기열에 등록합니다. 체크인 순서의 대기번호는 고정하고, 앞사람 취소·탑승·노쇼에 따라 `내 앞 대기 인원`만 실시간으로 갱신합니다. AI는 이번 버스와 다음 버스의 승차 가능성, 호출할 대기번호 구간, 호출 시점을 예측합니다. 호출 전 승객은 주변 그늘·쉼터·실내 공간에서 기다릴 수 있고, 호출된 승객만 기존 바닥 버스번호가 표시된 해당 노선 대기 위치로 이동하므로 전체 대기자가 계속 줄을 설 필요가 없습니다.

## LBS

- 사용자 현재 위치와 정류장 좌표 기반 100m 지오펜싱
- 정류장 도착 여부, 위치 이탈 여부, 체크인 유지 판단
- 버스 실시간 위치와 정류장 순번을 활용한 호출 시점 계산
- 주변 그늘, 쉼터, 지하철 출구 등 분산 대기 공간 안내로 확장 가능

## AI

QueueBus의 AI는 누가 먼저 탈지 정하지 않습니다. 순번은 위치 인증 기반 선착순으로 보장합니다. AI는 대기 수요, 차량별 잔여좌석, 사용자별 다음 버스 승차 가능성, 호출 인원·타이밍, 혼잡·기상 위험, 위치조작 의심 패턴을 예측합니다.

MVP는 규칙 기반/이동평균으로 시작하고, 데이터 축적 후 LightGBM, XGBoost, 시계열 예측, Isolation Forest 등으로 고도화합니다.

## 고객

1차 사용자는 출퇴근 광역버스 이용자, 고령자, 임산부, 교통약자, 초행길 이용자, 폭염·한파 취약 이용자입니다.

2차 고객은 지자체, 운수사, 교통 운영기관, 정류장 관리기관입니다. 이들은 QueueBus의 대기 수요 신호를 통해 배차 간격 조정, 예비차 투입 검토, 현장 안내 인력 배치, 혼잡 정류장 개선 우선순위를 판단할 수 있습니다.

## 사업모델

B2C 과금보다 B2G/B2B SaaS 모델이 적합합니다. 개인 이용자는 무료 또는 지자체 서비스로 제공하고, 운영기관이 정류장 단위 SaaS 이용료, 수요 분석 리포트, 운수사 수요 알림 API, 현장 키트 설치비, API 연동 비용을 부담합니다.

## PoC 계획

초기 PoC는 혼잡 광역버스 정류장 1곳과 노선 1~2개로 제한합니다. 퇴근 시간대 현장 관찰로 대기 인원, 탑승 인원, 미탑승 인원, 보행로 점유 여부를 수집하고, 웹 프로토타입으로 위치 인증, 고정 대기번호, 내 앞 대기 인원, 탑승 가능성 안내, 호출 대상 대기번호 구간, 운영자 대시보드를 시연합니다.

검증 지표는 위치 인증 성공률, 호출 응답률, 노쇼율, 탑승 가능성 예측 정확도, 평균 물리적 줄서기 시간 감소, 보행로 점유 완화, 사용자 만족도입니다.
