# 제출용 시각자료

생성 명령:

```bash
npm run submission:assets
```

생성 위치: `docs/assets/`

이 명령은 수정 가능한 SVG 원본을 재생성합니다. 현재 HWP 삽입용 PNG도 같은 폴더에 렌더링해 두었으며, 데이터 갱신 후에는 SVG를 먼저 다시 만든 뒤 PNG를 다시 내보냅니다.

## 2. 사용 원칙

- GBIS 차트는 `docs/15-gbis-evidence-report.md`와 `data/gbis-evening-station-buckets.csv`를 원천으로 생성합니다.
- 제출 직전 미니PC에서 최신 데이터를 가져온 뒤 `npm run gbis:derive`, `npm run gbis:analyze:evening`, `npm run gbis:evidence`, `npm run submission:assets` 순서로 다시 생성합니다.
- 현장조사 전에는 “다음 버스 승차 가능성과 호출 구간을 판단할 공식 API 예비 근거”로 표현하고, 현장조사 후 실제 대기 인원, 분산 대기 가능 공간, 미탑승 인원 수치를 같은 페이지에 함께 배치합니다.
- HWP에는 PNG를 우선 삽입하고, 원본 SVG는 수정 가능한 원본으로 보관합니다.

## 3. 캡션 초안

### 다음 버스 승차 가능성 판단 근거

> 경기도 GBIS 공식 API에서 M4137 차량별 잔여좌석을 수집한 결과, 서울 도심 후반 정류장은 시간대별 승차 가능성이 크게 달라졌다. QueueBus는 이 신호를 이용해 사용자가 줄에 계속 서 있지 않아도 이번 버스와 다음 버스의 승차 가능성을 판단하고, 호출 대상 대기번호 구간만 기존 대기 위치로 이동시키는 데 활용한다.

### 호출·다음차 안내 판단 지표

> 평일 퇴근 시간대 15분 단위 분석은 특정 정류장과 시간대에서 호출 인원을 줄이거나 다음차 안내를 해야 하는 구간을 보여준다. 같은 데이터는 운수사에 피크 수요 신호로 제공되어 배차 간격 조정 검토와 현장 운영 판단의 근거가 된다.

### 서비스 흐름도

> QueueBus는 좌석을 예약하는 서비스가 아니라, 정류장에 실제 도착한 승객을 위치 인증으로 대기열에 등록해 줄 대신 대기번호로 기다리게 하고, AI가 다음 버스 승차 가능성과 탑승 호출 구간을 예측하는 정류장 대기 관리 서비스다.

### 개인정보 보호 구조

> 운영자 대시보드는 개인의 실시간 위치나 이동 경로를 표시하지 않고, 정류장·노선·시간대 단위 집계 지표만 제공한다.
