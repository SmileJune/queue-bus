# LBS 스타트업 챌린지 심사 기준 대응표

본 문서는 2026 LBS 스타트업 챌린지 공식 모집요강의 아이디어 분야 평가 항목을 기준으로 QueueBus의 대응 논리를 정리합니다. 확인일은 2026-05-27이며, 최종 제출 전 공식 홈페이지의 최신 공고와 제출 서식을 다시 확인해야 합니다.

공식 모집요강상 아이디어 분야 1차 심사 항목은 창의성 및 혁신성 30점, 시장성 및 수요 20점, 기술 실현 가능성 20점, 사업화 가능성 20점, 사회적 가치 10점입니다. 창의성 및 혁신성 항목에는 AI 활용을 통한 서비스 개선 5점이 포함됩니다. 2차 심사는 아이디어 구체성, 시장 진입 전략, 기술 적용 계획, 팀 역량을 평가합니다.

## 1. 창의성 및 혁신성

| 평가 관점 | QueueBus 대응 |
| --- | --- |
| 기존 서비스와 차별화된 아이디어 | 좌석 예약이나 도착 알림이 아니라, 정류장 현장에 실제 도착한 승객의 대기 순서를 위치정보로 인증하고 혼잡을 관리합니다. |
| 오프라인 문제의 데이터화 | 물리적 줄서기, 보행로 점유, 새치기 갈등, 탑승 불확실성을 가상 대기열과 이벤트 데이터로 전환합니다. |
| 현장 UX 차별성 | 전체 대기자가 계속 줄 서는 것이 아니라, 고정 대기번호와 실시간 내 앞 대기 인원을 제공하고 이번 차량 탑승 가능성이 높은 대기번호 구간만 기존 노선 대기 위치로 호출합니다. |
| AI 활용 | AI는 순번을 정하지 않고 수요, 잔여좌석, 탑승 가능성, 호출 인원, 위험도를 예측해 서비스 운영을 개선합니다. |

강조 문장:

QueueBus는 좌석예약 서비스가 아닙니다. 정류장 현장에 실제로 도착한 승객의 대기 순서를 위치정보로 인증하고, AI로 탑승 수요와 호출 타이밍을 예측해 광역버스 정류장의 혼잡과 대기 불편을 줄이는 서비스입니다.

## 2. AI 활용

| AI 기능 | 심사 대응 포인트 |
| --- | --- |
| Demand Forecast AI | 정류장별·노선별·시간대별 대기 수요를 예측해 운영기관의 혼잡 대응 근거를 제공합니다. |
| SeatFlow AI | 차량의 현재 잔여좌석과 목표 정류장까지의 승하차 패턴으로 예상 탑승 가능 인원을 산정합니다. |
| Boarding Probability AI | 사용자의 대기번호, 내 앞 대기 인원, 예상 잔여좌석을 결합해 이번 차량 탑승 가능성을 안내합니다. |
| Call Optimizer AI | 예상 노쇼율과 최대 호출 인원을 반영해 호출 인원과 호출 시점을 계산합니다. |
| Congestion & Risk AI | 보행로 점유, 폭염/한파, 반복 만차 위험을 예측해 운영자 알림을 제공합니다. |

심사위원에게 명확히 전달해야 할 원칙은 `AI는 예측과 운영 최적화에만 사용하고, 순번은 위치 인증 기반 선착순으로 보장한다`는 점입니다.

## 3. 시장성 및 수요

| 평가 관점 | QueueBus 대응 |
| --- | --- |
| 고객 수요 | 출퇴근 광역버스 이용자는 장시간 줄서기, 탑승 불확실성, 날씨 노출 문제를 반복적으로 경험합니다. |
| 운영기관 수요 | 지자체와 운수사는 혼잡 정류장 관리, 배차 개선, 민원 대응에 필요한 정량 데이터가 부족합니다. |
| 확장성 | 초기에는 혼잡 정류장 1곳에서 시작하고, 같은 지자체 내 주요 광역버스 정류장으로 확장할 수 있습니다. |
| 반복 사용성 | 출퇴근 정류장 대기는 매일 반복되는 문제이므로 서비스 접점과 데이터 축적 빈도가 높습니다. |

## 5. 사업화 가능성

| 수익 모델 | 설명 |
| --- | --- |
| 지자체 PoC 구축비 | 혼잡 정류장 대상 초기 실증 구축 비용 |
| 정류장 단위 SaaS | 정류장별 대기열/혼잡 대시보드 월 이용료 |
| 운수사 데이터 구독 | 노선별 대기 수요, 만차 반복, 배차 개선 리포트 |
| 현장 키트 설치비 | QR 안내판, 기존 바닥 버스번호 연계 호출 안내, 전광판 연동 |
| API 연동/유지보수 | 교통 운영 시스템 연동 및 운영 지원 |

시장 진입은 민원이 잦은 혼잡 정류장의 문제를 작은 PoC로 검증하고, 정량 지표를 기반으로 지자체/운수사 확장 제안을 하는 방식이 적합합니다.

| 팀 역량 | 기획, 프론트엔드 PoC, 데이터 분석, 교통 운영 협력 역할 분담 |

## 8. 보완 필요 사항

- 최종 제출 전 공식 제출서류 파일명 규칙과 마감 시간을 재확인해야 합니다.
- 실제 후보 정류장 1곳의 현장 사진 또는 관찰 데이터를 추가하면 시장성과 구체성이 크게 강화됩니다.
- 팀 구성원 중 청년 요건, 위치정보/위치기반사업체 미보유 요건 등 아이디어 분야 참가 자격을 확인해야 합니다.
