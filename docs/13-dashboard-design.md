# GBIS 대시보드 설계

## 목적

대시보드는 수집기가 잘 돌고 있는지 확인하고, M4137의 출근/퇴근 방향별 잔여좌석과 좌석 변화 기반 탑승 추정치를 빠르게 읽기 위한 운영 화면입니다.

상시 수집 환경에서는 빌드된 React 앱보다 CSV를 직접 읽는 읽기 전용 서버가 더 적합합니다. 수집 CSV가 갱신될 때마다 서버가 최신 파일을 다시 읽으면 되므로 재빌드가 필요 없습니다.

## 화면 구성

1. 수집 상태

- 마지막 수집 시각
- 마지막 수집 경과 시간
- 최근 창의 차량 수
- 최근 창의 스냅샷 행 수
- 수집 중단 의심 상태

2. 방향 비교

- 출근 동탄→서울
- 퇴근 서울→동탄
- 방향별 대상 구간 안에서 관측된 평균/최소/최대 잔여좌석
- 방향별 탑승 추정 합계

위치 API 응답은 대상 정류장 수만큼 CSV에 반복 저장될 수 있으므로, 대시보드 표시용 방향 통계는 `collected_at + 차량` 기준으로 중복 제거한 샘플을 사용합니다.

3. 분석 준비도

- 평일 출근 피크: 06:30~09:30 KST
- 평일 퇴근 피크: 16:00~20:30 KST
- 휴일/주말 외출 피크: 10:00~14:00 KST
- 휴일/주말 복귀 피크: 16:00~20:00 KST
- 정류장별 피크 통과 샘플 수
- 정류장별 피크 데이터가 쌓인 날짜 수
- 목표 기준: 정류장별 30샘플 이상, 3일 이상

데이터가 쌓이는 동안 이 화면으로 분석 가능 여부를 판단합니다. `대기`는 아직 피크 통과 샘플이 없는 상태, `수집 중`은 일부 샘플이 있는 상태, `분석 가능`은 목표 샘플과 날짜 기준을 모두 만족한 상태입니다.

4. 정류장별 다음 차량

- 대상 정류장 순번
- 정류장별 접근 중인 차량
- 접근 차량의 남은 정류장 수
- 접근 차량의 잔여좌석
- 해당 정류장 순번을 실제 통과한 샘플의 평균 잔여좌석

5. 차량별 현재 상태

- 차량번호
- 현재 정류장 순번
- 잔여좌석
- 혼잡도 코드
- 상태 코드

6. 잔여좌석 타임라인

- 분 단위 평균 잔여좌석
- 분 단위 최소 잔여좌석
- 운행 차량 수 변화

7. 정류장별 잔여좌석 프로파일

- 가로축: 정류장 순번
- 세로축: 잔여좌석
- 차량별 선: 같은 차량이 구간을 지나며 남긴 최신 정류장별 좌석 샘플
- 최신 점: 검은 테두리로 강조
- 시간 처리: 상단 최근 창 필터와 차량별 최신 점으로 반영

출근은 동탄 승차 구간, 퇴근은 서울 도심 승차 구간을 분리해서 봅니다. 이 뷰는 어느 정류장 구간에서 좌석이 급격히 줄어드는지 확인하는 데 사용합니다.

8. 탑승 추정

- 좌석 감소량 기반 추정 탑승 인원
- 0석 도착으로 인한 수요 절단 여부
- 정류장별/방향별 집계

## 구현 방식

현재 구현은 `scripts/gbis-dashboard-server.mjs`입니다.

```bash
npm run gbis:dashboard
```

기본 URL:

```text
http://localhost:4175
```

API:

```text
GET /api/summary?windowHours=12
```

CSV 다운로드:

```text
GET /data/gbis-seat-snapshots.csv
GET /data/gbis-boarded-estimates.csv
```

## 운영 설계

미니PC에서는 수집기, 추정치 생성 타이머, 대시보드를 각각 systemd 서비스로 분리합니다.

```text
queuebus-gbis-collector.service  계속 수집
queuebus-gbis-derive.timer       10분마다 추정치 갱신
queuebus-gbis-dashboard.service  읽기 전용 웹 대시보드
```

수집기가 잠시 죽어도 대시보드는 마지막 CSV를 읽어 상태를 보여줍니다. 마지막 수집 경과 시간이 5분을 넘으면 중단 의심으로 표시합니다.

## 다음 고도화

- SQLite 적재: CSV가 커지면 `snapshots`, `boarding_estimates` 테이블로 전환합니다.
- 시간대 필터: 평일 출근 06:30~09:30, 평일 퇴근 16:00~20:30, 휴일/주말 외출 10:00~14:00, 휴일/주말 복귀 16:00~20:00만 필터링합니다.
- 캘린더 비교: 평일/주말, 요일별 평균 좌석과 만차 빈도를 비교합니다.
- 만차 이벤트: `remain_seat_count = 0` 또는 매우 낮은 좌석을 이벤트로 따로 표시합니다.
- 현장 관찰 결합: 수기 대기 인원 CSV와 좌석 변화 추정치를 같은 시간축에 겹칩니다.
