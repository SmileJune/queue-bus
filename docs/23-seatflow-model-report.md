## 5. 과적합 방지 설계

- 무작위 row split을 1차 검증으로 쓰지 않고, 최신 KST 날짜를 검증 데이터로 분리했습니다.
- 날짜별 교차검증으로 특정 검증 날짜에만 맞은 결과인지 점검했습니다.
- 트리 깊이와 leaf 최소 샘플 수를 제한했습니다.
- 규칙 baseline과 반드시 비교합니다.
- 같은 차량의 연속 스냅샷을 외우는 데이터 누수 가능성을 이후 차량 그룹 분리 검증으로 추가 점검합니다.
