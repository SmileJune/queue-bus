# QueueBus 차별화 문서

## 1. 포지셔닝

QueueBus는 광역버스 좌석 예약 서비스가 아닙니다. 또한 단순 버스 도착 알림 서비스도 아닙니다.

QueueBus의 포지션은 `위치 인증형 현장 대기열 + 다음 버스 승차 가능성 안내 + 운수사 운영 수요 신호`입니다.

핵심 문장:

QueueBus는 좌석예약 서비스가 아니라, 정류장 현장에 실제 도착한 승객의 대기 순서를 위치정보로 인증해 줄 대신 대기번호로 편하게 기다리게 하고, AI로 다음 버스 승차 가능성과 호출 타이밍을 안내하는 정류장 대기 관리 서비스입니다. 축적된 대기 수요는 운수사와 운영기관의 배차 간격 조정 판단 근거로 제공됩니다.

## 2. 기존 좌석예약 서비스와의 차이

| 구분 | 기존 좌석예약 서비스 | QueueBus |
| --- | --- | --- |
| 목적 | 특정 차량의 좌석을 사전에 확보 | 현장 대기 순서와 정류장 혼잡 관리 |
| 이용 조건 | 예약 가능한 시간/차량 선택 | 정류장 반경 내 실제 도착 후 체크인 |
| 핵심 가치 | 좌석 확보 | 편한 대기, 다음 버스 승차 가능성 판단, 물리적 줄서기 완화 |
| 순번 기준 | 예약 규칙 또는 결제/예약 시점 | 위치 인증 완료 시점 선착순 |
| 현장 UX | 예약자 탑승 확인 중심 | 고정 대기번호와 내 앞 대기 인원을 보고 기존 노선 대기 위치로 이동 |
| 데이터 가치 | 예약 수요 데이터 | 정류장별·노선별·시간대별 실제 대기 수요와 배차 판단 신호 |
| 고객 | 개별 승객 중심 | 승객 + 지자체/운수사/운영기관 |

좌석예약은 특정 차량의 좌석을 미리 확보하는 서비스입니다. 반면 QueueBus는 정류장에 이미 도착한 승객의 현장 대기 순서를 관리합니다. 따라서 특정 좌석을 보장한다고 표현하지 않으며, 버스 기사에게 강제 정차나 좌석 배정을 요구하지 않습니다.

## 3. 버스 도착 알림 서비스와의 차이

| 구분 | 버스 도착 알림 | QueueBus |
| --- | --- | --- |
| 제공 정보 | 버스 도착 예정 시간 | 도착 예정 + 내 탑승 가능성 + 내 앞 대기 인원 |
| 대기열 관리 | 없음 | 노선별 가상 대기열 |
| 위치 인증 | 선택적 또는 없음 | 서비스 작동 조건 |
| 혼잡 완화 | 정보 제공에 그침 | 호출 인원과 호출 타이밍 조절 |
| 운영 데이터 | 조회/알림 중심 | 체크인, 호출, 탑승, 노쇼 이벤트 축적 |
| 운영자 가치 | 제한적 | 대기 수요 기반 배차 간격 조정과 현장 운영 근거 제공 |

도착 알림은 `버스가 언제 오는지`를 알려줍니다. QueueBus는 `내가 이번 차 또는 다음 차를 탈 가능성이 있는지`, `호출 전까지 어디서 편하게 기다릴 수 있는지`, `언제 기존 노선 대기 위치로 이동해야 하는지`, `운수사가 미리 봐야 할 대기 수요가 얼마나 되는지`까지 다룹니다.

## 4. QueueBus만의 핵심 차별점

### 위치 인증형 선착순 대기열

사용자가 정류장 반경 내에 실제로 도착해야 체크인할 수 있습니다. 원격 사전 예약이나 대리 줄서기를 줄이고, 현장 도착자 중심의 공정한 대기열을 구성합니다.

### AI는 순번이 아니라 운영을 최적화

AI가 먼저 탈 사람을 정하지 않습니다. 순번은 위치 인증 시점 기준 선착순이며, AI는 예상 탑승 가능 인원, 사용자별 탑승 가능성, 호출 인원, 호출 시점, 혼잡 위험을 계산합니다.

### 고정 대기번호와 기존 노선 대기 위치의 연결

가상 대기열만 있으면 버스 도착 직전 다시 혼잡해질 수 있습니다. QueueBus는 기존 바닥 버스번호가 표시된 노선별 대기 위치를 그대로 활용하되, 체크인 시 부여한 대기번호는 고정하고 내 앞 대기 인원만 실시간 갱신합니다. 호출 전에는 줄에 계속 서지 않아도 되고, 버스 도착 전에는 이번 차량 탑승 가능성이 높은 대기번호 구간만 기존 노선 대기 위치로 호출합니다.

### 공공 운영 데이터 축적

QueueBus는 사용자가 체크인할수록 정류장별·노선별·시간대별 대기 수요 데이터가 축적됩니다. 이 데이터는 AI 기반 승차 가능성 예측뿐 아니라 운수사의 배차 간격 조정, 예비차 투입 검토, 지자체의 정류장 운영 개선 근거가 됩니다.

### B2G/B2B 확장성

개인 이용자를 무료로 확보하고, 지자체/운수사/교통 운영기관에 정류장 단위 SaaS, 대시보드, 수요 분석 리포트, 현장 키트 설치를 제공하는 구조로 사업화할 수 있습니다.

## 5. 경쟁 우위 확보 전략

- 혼잡 정류장 1곳의 실제 관찰 데이터로 문제를 구체화합니다.
- 웹 프로토타입으로 사용자와 운영자 흐름을 빠르게 시연합니다.
- 위치정보와 개인정보 보호 원칙을 초기 문서부터 명확히 제시합니다.
- 공식 버스정보 API 연동 전에도 mock 데이터와 수기 관찰로 PoC를 진행할 수 있게 설계합니다.
- 공모전 이후 BM 컨설팅, 기술 멘토링, AI 전환 컨설팅을 통해 예측 모델과 사업화 전략을 보완합니다.
